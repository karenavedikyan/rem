# MVP: Навигатор ремонта RemCard

## Цель

Сделать веб-инструмент, который за 3–7 ответов клиента формирует базовый маршрут ремонта:

- этапы работ,
- рекомендуемые типы исполнителей,
- категории материалов/услуг,
- советы и типичные ошибки.

Навигатор не бронирует мастеров автоматически. Он готовит понятный план и отправляет заявку в RemCard с пометкой «Навигатор ремонта».

## Формат

- Отдельная страница: `/navigator/`
- Два экрана:
  1. Вопросы (форма)
  2. Результат (шаги маршрута + отправка заявки)
- Серверные эндпоинты:
  - `POST /api/navigator-route` — генерация маршрута (AI + fallback)
  - `POST /api/navigator-submit` — отправка заявки по маршруту в канал RemCard

## Входные данные (MVP)

Обязательные поля:

1. Тип объекта: квартира / дом / коммерческое помещение
2. Статус объекта:
   - новостройка без отделки
   - новостройка с базовой отделкой
   - вторичка, частичный ремонт
   - вторичка, полный ремонт
3. Стадия:
   - только планирую
   - замеры/планировка
   - черновые
   - чистовая
   - мебель/свет
4. Бюджет: диапазоны + вариант «пока не знаю»
5. Срок старта: сейчас / месяц / 3 месяца / позже

Дополнительно:

- особенности объекта (текст),
- имя,
- контакт (телефон/Telegram).

## Выходные данные

Маршрут формируется как массив `steps[]`, где каждый шаг содержит:

- `id`
- `title`
- `description`
- `stage_type`
- `recommended_professionals[]`
- `recommended_categories[]`
- `tips[]`
- `resources[]`

Базовый целевой формат:

```json
{
  "steps": [
    {
      "id": "step_1",
      "title": "Планирование и замеры",
      "description": "Определяем задачи ремонта, делаем базовые замеры и планировку помещения.",
      "stage_type": "planning",
      "recommended_professionals": ["дизайнер", "прораб", "мастер-универсал"],
      "recommended_categories": ["замеры и планировка", "дизайн-проект (по желанию)"],
      "tips": [
        "Сфотографируйте помещение с разных ракурсов до начала работ.",
        "Зафиксируйте текущие размеры и особенности."
      ],
      "resources": [
        {
          "type": "video",
          "title": "Как правильно подготовиться к ремонту",
          "url": "https://example.com/video-1"
        }
      ]
    }
  ]
}
```

## Логика MVP (текущая реализация)

`/api/navigator-route` работает в двух режимах:

1. **AI-режим** (если настроен `REMCARD_OPENAI_API_KEY`/`OPENAI_API_KEY`)  
   Сервер отправляет ответы пользователя в модель и получает `steps[]` в целевом JSON-формате.
2. **Fallback-режим** (если AI недоступен)  
   Маршрут собирается из шаблонов с адаптацией под объект/стадию/бюджет/сроки.

В любом режиме:

- учитывается статус объекта (например, частичный ремонт сокращает этапы),
- учитывается текущая стадия (маршрут стартует с релевантного шага),
- добавляются подсказки по бюджету, срочности и особенностям объекта,
- формируется 3–6 шагов,
- используется единая база `knowledge/knowledge-base.json` (шаблоны этапов + знания по стадиям + контент для страницы базы знаний).

Это даёт один источник данных одновременно для:

- `api/navigator-route` (AI-промпт и fallback-маршрут),
- `navigator/navigator.js` (fallback на фронте),
- `navigator/navigator-knowledge.js` (встроенный блок базы знаний внутри `/navigator`).

Локальная проверка структуры knowledge base:

```bash
node validate-knowledge-base.mjs
```

Типовые шаги:

1. Планирование и замеры
2. Черновые работы
3. Инженерные работы (электрика/сантехника)
4. Чистовая отделка
5. Мебель, свет и декор

## Интеграция с RemCard

Кнопка под результатом:

**«Отправить заявку в RemCard по этому маршруту»**

Отправляет в тот же канал, что и остальные заявки, текст с:

- параметрами объекта,
- стадией, бюджетом, сроком,
- списком шагов маршрута,
- контактом клиента,
- меткой «Навигатор ремонта».

## Ограничения MVP

- Нет личного кабинета и сохранения маршрутов.
- Рекомендации по типам специалистов и категориям, без динамической цены.
- Ресурсы/лайфхаки — базовые (шаблон), будут расширяться.

## Переменные окружения (Vercel)

Для AI-генерации:

- `REMCARD_OPENAI_API_KEY` (или `OPENAI_API_KEY`)
- `REMCARD_OPENAI_MODEL` (опционально, по умолчанию `gpt-4o-mini`)

Для отправки маршрута в Telegram:

- `REMCARD_TELEGRAM_BOT_TOKEN` (или `TELEGRAM_BOT_TOKEN`)
- `REMCARD_TELEGRAM_CHAT_ID` (или `TELEGRAM_CHAT_ID`)
