# Процесс Telegram -> GitHub -> сайт (партнёры RemCard)

Этот процесс нужен, чтобы заявки из Telegram обрабатывались дисциплинированно и прозрачно.

## 1) Получить заявку в Telegram

- Заявка приходит в группу/чат `RemCard заявки`.
- Менеджер копирует данные заявки (имя, тип, город, специализация, контакты).

## 2) Создать GitHub Issue по шаблону

- В репозитории откройте `New issue`.
- Выберите шаблон: **Partner application (Telegram)**.
- Заполните поля и создайте issue.

Рекомендуемые label'ы:

- `partner-request` — новая заявка
- `approved` — заявка проверена и одобрена

Шаблон issue автоматически ставит `partner-request`.

> Если label'ов ещё нет в репозитории, создайте их в настройках Labels вручную.

## 3) Модерация заявки

В issue:

- проверьте контакты/описание,
- отметьте чекбокс "Проверка данных выполнена",
- при одобрении добавьте label `approved`.

После этого issue можно закрыть (или оставить открытой до публикации — по вашему процессу).

## 4) Публикация на сайт

Публикация может идти двумя путями:

1. Через Partners API (`POST /partners` c `isApproved: true`)
2. Через fallback JSON (`confirmed-partners.json`) при недоступности API

После публикации отметьте чекбокс "Партнёр опубликован на сайте" и закройте issue.

## 5) Минимальная проверка результата

- `GET /partners?approved=true` возвращает партнёра
- партнёр виден в блоке на сайте RemCard
